This week we shall try to do the same with another data structure known as queues.
A Queue is a linear structure which follows a particular order in which the operations are performed. The order is First-In-First-Out (FIFO). A good example of a queue is any queue of consumers for a resource where the consumer that came first is served first.
The difference between stacks and queues is in removing. In a stack we remove the item the most recently added; in a queue, we remove the item the least recently added.It contains two pointers - front pointer pointing to the first element of the queue and rear pointer pointing to the last element in the queue. Whenever an element is added in the queue, it is added at the end of the queue, and an element can be deleted only from the front of the queue.

Properties of Queues


Some key properties related to queue are -

  1. 1. A queue can be defined as an ordered list which enables insert operations to be performed at one end called REAR and delete operations to be performed at another end called FRONT.
  2. 2. A Queue is an abstract data type with a pre-defined capacity, which means that it can store the elements of a limited size.
  3. 3. It is a data structure that follows some order to insert and delete the elements, and that order can be LIFO or FILO. This ordering is maintained by 2 operations - enqueue and dequeue.



Applications of Queues


Some key applications related to queue are -

  1. 1. Queues are widely used as waiting lists for a single shared resource like printer, disk, CPU.
  2. 2.Queues are used in asynchronous transfer of data.
  3. 3. Queues are used as buffers in most of the applications like MP3 media player, CD player, etc.
  4. 4. Queues are used in operating systems for handling interrupts.



Operations on Queues


There are some basic operations that allow us to perform different actions on a queue.

  1. 1. Enqueue : Add an element to the end of the queue.
  2. 2. Dequeue : Remove an element from the front of the queue.
  3. 3. IsEmpty : Check if the queue is empty.
  4. 4. IsFull : Check if the queue is full
  5. 5. Peek : Get the value of the element at the front of the queue without removing it
